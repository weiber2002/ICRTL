TASK: Implement a synthesizable Verilog module "DT" that reads a binary 128x128 image from "sti\_ROM", computes the 8-distance (chessboard) transform using forward and backward passes, and writes an 8-bit 128x128 result image to "res\_RAM". When all results are written, assert done=1. Do not change any port names or widths.

1. I/O (fixed)

* input  clk            : posedge-synchronous clock
* input  reset          : active-low asynchronous reset (Low=reset). Testbench holds Low for \~2 cycles, then deasserts to High.
* output sti\_rd         : sti\_ROM read enable (serviced at negedge)
* output \[9:0]  sti\_addr: sti\_ROM address 0..1023 (<= 1 address serviced per cycle; keep stable across the servicing negedge)
* input  \[15:0] sti\_di  : sti\_ROM read data (16 packed binary pixels)
* output res\_wr         : res\_RAM write enable (write at posedge)
* output res\_rd         : res\_RAM read enable (read at negedge)
* output \[13:0] res\_addr: res\_RAM address 0..16383 (row-major result addresses)
* output \[7:0]  res\_do  : res\_RAM write data (grayscale)
* input  \[7:0]  res\_di  : res\_RAM read data
* output done           : High once after all result pixels are written

2. Memory mapping and timing

* sti\_ROM (input):

  * Image size: 128x128 = 16384 pixels, 1 bit per pixel.
  * ROM geometry: width 16 bits, depth 1024.
  * Packing: address a holds pixels \[16a .. 16a+15] in row-major order (left-to-right within a row, top-to-bottom across rows).
  * Timing: on the clock negedge, if sti\_rd=1, sti\_di updates immediately with the word at sti\_addr. At most one ROM address serviced per cycle. Keep sti\_addr stable across the servicing negedge.
* res\_RAM (output):

  * Result size: 128x128 = 16384 pixels, 8 bits per pixel, row-major at addresses 0..16383.
  * Timing:

    * READ: if res\_rd=1 at a negedge, res\_di immediately reflects the word at res\_addr.
    * WRITE: if res\_wr=1 at a posedge, write res\_do to res\_addr.
  * In a given cycle, you may perform one read (negedge) and one write (posedge).
  * res\_RAM may be used as scratch during computation.

3. Algorithm (8-distance / chessboard transform)

* Input semantics: object pixels are 1, background pixels are 0. Test data guarantee the object region does not touch the outermost border ring.
* Initialization:

  * For each pixel p(x,y): value = 1 if object (input bit=1), else 0 (background).
* Forward pass (scan left->right, top->bottom):

  * For each object pixel p(x,y):

    * let m = min( NW, N, NE, W )
    * update p(x,y) = m + 1
  * Background pixels keep value 0.
* Backward pass (scan right->left, bottom->top):

  * For each object pixel p(x,y):

    * let r = min( E, SE, S, SW )
    * update p(x,y) = min( p(x,y), r + 1 )
* After the backward pass, p(x,y) is the chessboard distance to the nearest background pixel. Background pixels remain 0.

4. Control and finish

* While reset=0: clear all internal state; do not read or write memories; done=0.
* After reset transitions to 1: fetch binary words from sti\_ROM and execute the forward and backward passes (you may read/write res\_RAM as scratch if desired).
* Write all 16,384 result pixels to res\_RAM in row-major order (writes occur at posedges when res\_wr=1).
* After the last write, assert done=1 (High). The host will validate and end the run.

5. Constraints

* Keep all ports, widths, and edge-sensitive protocols exactly as specified.
* sti\_ROM: at most one address serviced per cycle; hold sti\_addr stable across the servicing negedge.
* res\_RAM: at most one read (negedge) and one write (posedge) per cycle.
* Any internal micro-architecture is acceptable (buffering, streaming, tiled, etc.) as long as behavior and external protocols match this specification.
