TASK: Implement a synthesizable Verilog module huffman that follows only the I/O, timing, ordering/tie-break, packing, and determinism contracts below (no constraints on internal micro-architecture). All ports are 1-D flattened. The design must compute Huffman codes from the streamed input—no exhaustive lookup or precomputed tables.

# 1) Goal & Scope

- Input: exactly 100 grayscale pixels, one per cycle while gray_valid=1. Each pixel is in the set {0x01..0x06}, representing symbols A1..A6.

- Output phases:
  - Counts phase: total occurrences for A1..A6 (8-bit each), flagged by a single-cycle CNT_valid.
  - Codes phase: per-symbol Huffman code and mask (each 8-bit), flagged by a single-cycle code_valid.

- Required algorithm: build Huffman codes deterministically from the 100-sample statistics using the exact ordering and tie-breaking rules in §4.
- Forbidden: exhaustive enumeration, prebuilt mapping tables, or any solution that does not construct codes from the measured counts.

# 2) Top-Level Ports (names, widths, meaning)

- input clk // posedge-synchronous
- input reset // Asynchronous, active-high; while 1: clear state, deassert all valids
- input gray_valid // when 1, one pixel is present on gray_data this cycle
- input [7:0] gray_data // pixel ∈ {8'h01..8'h06} = A1..A6
- output CNT_valid // 1-cycle pulse; counts are valid on CNT1..CNT6 this cycle
- output [7:0] CNT1 // count of A1 (0..100)
- output [7:0] CNT2 // count of A2
- output [7:0] CNT3 // count of A3
- output [7:0] CNT4 // count of A4
- output [7:0] CNT5 // count of A5
- output [7:0] CNT6 // count of A6
- output code_valid // 1-cycle pulse; codes/masks valid on HC*/M* this cycle
- output [7:0] HC1, HC2, HC3, HC4, HC5, HC6 // per-symbol Huffman codes, packed (see §4.4)
- output [7:0] M1, M2, M3, M4, M5, M6 // per-symbol masks, which bits of HC* are valid (see §4.4)

# 3) Handshake & Global Timing

(1) Reset & Ready
- While reset=1: clear all internal state; do not assert CNT_valid or code_valid.
- After reset deasserts (1→0), the core is ready to accept input.

(2) Input streaming
- The testbench asserts gray_valid=1 and streams 100 pixels, exactly one per cycle on gray_data.
- Sample inputs at posedge clk whenever gray_valid=1.
- After the 100th pixel, gray_valid goes Low. There are no other handshakes or back-pressure signals.

(3) Output phases
- Counts phase (T4): when all six counts are ready, assert CNT_valid=1 for one cycle and drive CNT1..CNT6.
- Codes phase (T5): when all Huffman codes are ready, assert code_valid=1 for one cycle and drive HC1..HC6 and M1..M6.
- All outputs are synchronous to clk. Valid strobes (CNT_valid, code_valid) are single-cycle pulses.

# 4) Algorithmic Contract (deterministic Huffman)
4.1 Symbols & Indices
- Symbols: A1..A6 correspond to values 0x01..0x06. Use indices 1..6 in all tie-breaks (“smaller index” = “smaller number”).

4.2 Three Stages (conceptual)
- Initialization (sort): Sort the six symbols by descending count. If counts tie, the smaller index is higher in the list.
- Combination rounds: Repeatedly merge the bottom two list items (the two lowest counts) into one group with count = sum of members. Reinsert the merged group and resort in descending count.
- Tie rule in combination: if the merged group’s count ties others, place the merged group last within that tie group (i.e., at the lowest position among equals). Continue until only two items remain.
- Split / Code assignment: Starting from the final 2-item list, back-propagate codes to leaves. At each split:

Upper item gets bit ‘0’
Lower item gets bit ‘1’
Codes are formed by appending the split bit at each step while moving from parent to children until reaching each symbol.
Tree view (if you draw one): for any 2-child node, place the smaller count on the left, larger on the right. This matches the upper=‘0’, lower=‘1’ rule when using list splits.

4.3 Probabilities
- Probabilities are proportional to counts (100 samples total). You may order directly by counts—no normalization required.

4.4 Code Packing to Outputs (HC*/M*)
- Let a symbol’s Huffman code have length L (1..8).
- Mask rule: Mx[L-1:0] = 1 and Mx[7:L] = 0.
- Code packing: Place the code left-to-right into HCx[L-1:0]. Force HCx[7:L] = 0.
- Example: code 00 → HC=8'h00, M=8'h03.
- Example: code 01010 → HC=8'h0A, M=8'h1F.
(The numeric value of HCx[L-1:0] equals the binary code read left→right.)

4.5 Prohibited Approach
- Not allowed: exhaustive enumeration / precomputed mapping. You must execute initialize → repeated combination (with tie rules) → split on the observed counts.

# 5) Determinism & Tie-Breaking (must-follow)
- Init sort: descending count; tie → smaller index earlier.
- Combination: always merge the bottom two; reinsert & resort; tie on reinsert → merged group placed last among equals.
- Split: upper→‘0’, lower→‘1’; append bits while descending to leaves.
- Packing: pack bits and masks exactly as in §4.4.

# 6) Verification Expectations (checker assumptions)

- Inputs are restricted to {0x01..0x06}. Exactly 100 samples per run.
- The checker latches counts on the cycle with CNT_valid=1, and latches codes/masks on the cycle with code_valid=1.
- Code equivalence is determined solely by the deterministic procedure and tie rules above.
